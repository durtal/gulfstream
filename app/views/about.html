About

This web app contains ratings for individual horses who have raced at Gulfstream Park from December 2012 onwards.

Ratings have been created using data collected from a variety of sources. With the data collected, the bulk of the work has been carried out using the R programming language and a package I created called RcappeR, which is now on github for others to use, there are help pages too. I hope that ratings found here, and posts that will walk through my handicapping process will encourage others to use the package, offer suggestions, improvements, alternatives, and to share with other racing fans.

It was June 2013 that I started to learn R on my old wordpress blog, I also knew very little HTML and CSS, just enough to highlight some clumsy R code (when I used to use attach()). At the time I had no idea I would end up with a web app like this (which is still very much a work in progress). I have taken advantage of a lot of open source software in the creation, storage, saving, building, and presentation of the ratings/analysis found in these pages. The primary software I have used includes:

  • R - programming language, I use RStudio,
  • Git - version control, Github also hosts this app
  • MongoDB - database
  • AngularJS - web app framework
  • Yeoman - web app scaffolding tool, uses 2 more open source tools, GruntJS and Bower

I also owe a couple of people a word of thanks, whether they knowingly, or unknowingly helped me, these are (in no particular order):

  • Hadley Wickham (@hadleywickham) - author of many great R packages, some of which are vital to my RcappeR package, and the many excellent tutorials on building packages.
  • Simon Rowlands (@rowleyfiletf) - many articles have helped build my interest in racing, my ratings utilise a method of race standardisation explained by Simon, which is included in my RcappeR package.
  • Bob Wilkins - author of Bioenergetics and Racehorse Ratings an excellent, but challenging, book.

  • race previews

Get Involved

If anyone is interested in contributing, has any questions, ideas for features, visualisations, don't hesitate to get in touch via twitter or email (both listed below).

Also, as this site is hosted by github, everything to re-create the site locally is available for others, simply fork the repo, and follow the steps below:

  • Install NodeJS
  • Open terminal or command prompt
  • navigate to directory of the forked repo (should be called gulfstream)
  • run npm install -g bower
  • run npm install -g grunt-cli
  • run npm install
  • run bower install
  • then finally run grunt serve and a browser should pop up with the app

If you encounter any errors then I'll do my best to figure out what's going wrong, but as I only started working with NodeJS, yeoman and bower about a month ago I may not be much help, apologies. This app also hasn't been tested using Karma (need to learn how to do that!).
